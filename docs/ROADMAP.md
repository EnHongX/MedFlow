# 路线图

## 阶段 0：项目骨架

- 建立目录结构
- 固定 4 个端的职责
- 建立产品范围文档
- 建立 API 和数据模型草案

## 阶段 1：MVP 闭环

- 患者端完成预约、取消、改约、状态查看
- 前台端完成到诊、叫号、分诊、过号处理
- 医生端完成预约查看、接诊、病人记录查看
- 管理后台完成科室、排班、号源、规则维护

验收标准：

- 患者能完成一次有效预约。
- 前台能把预约患者变成候诊队列。
- 医生能接诊并结束一次就诊。
- 管理员能配置支撑上述流程的基础数据。
- 不要求服务拆分和端口规划。

## 阶段 2：规则完善

- 取消/改期规则
- 停诊/加号规则
- 爽约处理
- 队列过号策略
- 通知提醒
- 操作记录

## 阶段 3：运营与稳定性

- 统计面板
- 审计日志
- 权限细化
- 异常补偿
- 部署与监控

## 暂缓

- 医保
- 支付
- 处方
- 检查检验
- 复杂 BI

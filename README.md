# MedFlow

MedFlow 是一个面向门诊场景的多角色医疗预约与分诊协同系统。

目标不是单纯“挂号”，而是把患者预约、前台分诊、医生接诊、管理员配置这些流程串起来：

```text
患者发起预约 -> 系统分配号源/时段 -> 前台处理到诊与分诊 -> 医生接诊 -> 后台维护规则
```

## 角色

- 患者
- 医生
- 前台/导诊台
- 管理员

## 端

- 患者端：患者注册登录、预约、改约、取消、查看状态和排队进度。
- 医生端：查看预约列表、患者信息、接诊状态和就诊记录。
- 前台端：处理签到、分诊、叫号、过号和候诊队列。
- 管理后台：维护科室、医生、排班、号源、停诊和预约规则。

## 应用划分

项目现阶段只按 4 个端组织，不拆服务端口：

| 应用 | 目录 | 使用者 | 核心职责 |
| --- | --- | --- | --- |
| 患者端 | `apps/patient-web` | 患者 | 预约、取消、改约、查看状态和排队进度 |
| 医生端 | `apps/doctor-web` | 医生 | 查看预约、接诊、查看病人记录 |
| 前台端 | `apps/frontdesk-web` | 前台/导诊台 | 到诊、叫号、分诊、过号处理 |
| 管理后台 | `apps/admin-web` | 管理员 | 科室、排班、号源、规则维护 |

## 项目结构

```text
backend/              # NestJS 统一后端 API
apps/                 # 各角色前端应用
docs/                 # 产品、架构、接口、数据模型文档
```

## 技术栈

| 层 | 选择 |
| --- | --- |
| 后端 | `Node.js + TypeScript + NestJS` |
| ORM | `Prisma` |
| 数据库 | `PostgreSQL` |
| 前端 | `Vue 3 + Vite + TypeScript + Element Plus` |
| 权限 | 基于角色的 RBAC |

本地端口已固定：

| 应用 | 端口 |
| --- | --- |
| 后端 API | `5100` |
| 患者端 | `5110` |
| 医生端 | `5120` |
| 前台端 | `5130` |
| 管理后台 | `5140` |
| PostgreSQL | `5432` |

## 本地启动

安装依赖：

```bash
npm install
```

启动后端：

```bash
npm run dev:backend
```

启动前端：

```bash
npm run dev:patient
npm run dev:doctor
npm run dev:frontdesk
npm run dev:admin
```
